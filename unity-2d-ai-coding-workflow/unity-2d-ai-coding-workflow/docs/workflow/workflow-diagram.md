## 渐进式披露详细流程

```
用户提出需求
    │
    ▼
AI识别信息缺口
    │
    ├─ 需求确认缺口？
    │   └─ 是 → 提出澄清问题
    │           │
    │           └─ 用户回答
    │                   │
    ├─ 技术细节缺口？←─┘
    │   └─ 是 → 提出澄清问题
    │           │
    │           └─ 用户回答
    │                   │
    ├─ 设计决策缺口？←─┘
    │   └─ 是 → 提出澄清问题
    │           │
    │           └─ 用户回答
    │                   │
    ├─ 实现细节缺口？←─┘
    │   └─ 是 → 提出澄清问题
    │           │
    │           └─ 用户回答
    │                   │
    └─ 信息完整？←─────┘
        │
        ├─ 否 → 继续识别缺口
        │
        └─ 是 → 执行开发任务
```

## 任务分级决策树

```
任务开始
    │
    ▼
代码行数 < 100？
    │
    ├─ 是 ─┐
    │       ▼
    │    单一职责？
    │       │
    │       ├─ 是 ──→ A级任务（完全自动化）
    │       │          ├─ 使用A级Skills
    │       │          └─ 效率提升80%+
    │       │
    │       └─ 否 ─┐
    │               ▼
    │            需理解上下文？
    │               │
    │               ├─ 是 ──→ B级任务（半自动化）
    │               │          ├─ 使用B级Skills
    │               │          └─ 效率提升50-70%
    │               │
    │               └─ 否 ──→ A级任务
    │
    └─ 否 ─┐
            ▼
         涉及性能优化？
            │
            ├─ 是 ──→ C级任务（AI辅助）
            │          ├─ 人工主导+AI辅助
            │          └─ 效率提升30-50%
            │
            └─ 否 ─┐
                    ▼
                 跨模块影响？
                    │
                    ├─ 是 ──→ B级任务（半自动化）
                    │
                    └─ 否 ──→ A级任务
```

## Skills使用决策

```
开发阶段
    │
    ├─ 开发前（分析）⚠️ 强制
    │   ├─ @architecture-design-analyzer（必须）
    │   └─ @impact-scope-analyzer（必须）
    │
    ├─ 开发前（技术文档）
    │   ├─ 物理问题 → @unity-2d-physics (L1级)
    │   ├─ 精灵问题 → @unity-2d-sprite (L1级)
    │   ├─ 动画问题 → @unity-2d-animation (L1级)
    │   ├─ 地图问题 → @unity-tilemap (L1级)
    │   ├─ 输入问题 → @unity-input-system (L1级)
    │   ├─ UI问题 → @unity-ui-system (L1级)
    │   └─ 音频问题 → @unity-audio-system (L1级)
    │
    ├─ 开发中（编码）
    │   ├─ 角色控制 → @unity-2d-character-controller (L2级)
    │   ├─ 数据配置 → @unity-scriptableobject-config (L2级)
    │   ├─ 场景管理 → @unity-scene-management (L2级)
    │   ├─ 对象池 → @unity-object-pool (L2级)
    │   ├─ 存档系统 → @unity-save-system (L2级)
    │   ├─ UI面板 → @unity-ui-panel (A级)
    │   └─ 管理器 → @unity-singleton-manager (A级)
    │
    └─ 开发后（审查）⚠️ 强制
        └─ @unity-code-review（必须）
```

## 审查决策流程

```
审查开始
    │
    ▼
功能正确性？
    │
    ├─ ❌ → 必须修改
    │        └─ 重新生成或手动修改
    │
    └─ ✅ ─┐
            ▼
         Unity最佳实践？
            │
            ├─ ❌ → 必须修改
            │
            └─ ✅ ─┐
                    ▼
                 代码质量？
                    │
                    ├─ ⚠️ → 警告（可接受但需改进）
                    │
                    ├─ ❌ → 必须修改
                    │
                    └─ ✅ ─┐
                            ▼
                         安全性？
                            │
                            ├─ ❌ → 必须修改
                            │
                            └─ ✅ ──→ 审查通过
```
