# 架构设计分析器

## 目的

在开发前分析架构设计和影响面，评估改动范围和风险，避免盲目开发，生成完整的分析报告供审批。

## 使用场景

**强制使用场景**：
- ✅ 所有新功能开发
- ✅ 修改现有核心系统
- ✅ 性能关键路径改动
- ✅ 跨模块的功能

**可选使用场景**：
- ⚠️ 简单Bug修复（< 10行代码）
- ⚠️ 配置文件调整
- ⚠️ UI文案修改
- ⚠️ 纯资源替换

## 工作流程

### 阶段1：系统定位分析

**操作**：自动分析系统归属
- 识别所属模块
- 分析模块关系（上游依赖、下游影响、横向关联）
- 生成模块依赖图

**输出示例**：
```
系统定位：
- 所属模块：物品系统
- 上游依赖：数据管理、UI系统
- 下游影响：存档系统
- 横向关联：战斗系统（物品使用）

模块归属是否正确？
```

**确认点**：请用户确认模块归属是否正确。

### 阶段2：设计决策建议

**操作**：提供设计方案建议
- 推荐设计模式及理由
- 备选方案及优缺点对比
- 数据流向设计
- 扩展性建议

**输出示例**：
```
推荐模式：MVC + 观察者模式
理由：
- 数据与展示分离，便于维护
- 观察者模式实现物品变化通知
- 符合项目现有架构风格

备选方案：
1. MVVM模式 - 数据绑定自动化，但Unity支持不完善
2. ECS模式 - 性能最优，但学习曲线陡峭

是否采用推荐方案？
```

**确认点**：请用户选择设计方案。

### 阶段3：影响面评估

**操作**：分析改动影响
- 需要修改的现有文件列表
- 依赖关系梳理
- 性能影响评估
- 兼容性检查

**输出示例**：
```
需要修改的文件：
- InventoryData.cs - 新增背包数据类
- UIManager.cs - 添加背包面板管理
- SaveManager.cs - 集成背包存档

性能影响：
- 内存：每个物品约100字节，100个物品≈10KB
- CPU：拖拽计算，每帧约0.1ms
- 风险：低

是否继续？
```

**确认点**：请用户确认是否继续或调整方案。

### 阶段4：风险识别

**操作**：识别潜在风险
- 潜在风险点
- 风险概率和影响
- 应对措施建议
- 技术债务预警

**输出示例**：
```
风险识别：
| 风险点 | 概率 | 影响 | 应对措施 |
|-------|------|------|---------|
| 拖拽性能 | 中 | 中 | 使用对象池优化 |  
| 存档数据丢失 | 低 | 高 | 自动备份机制 |
| UI层级冲突 | 低 | 低 | 统一UI管理器 |

技术债务预警：
- ☑️ 需要更新架构文档
- ☐ 引入新技术债务
- ☐ 需要重构现有代码

风险是否可接受？
```

**确认点**：请用户确认风险可接受。

### 阶段5：生成分析报告

**操作**：生成完整的架构设计与影响面分析报告
- Markdown格式
- 包含决策审批签名区
- 自动存档至`docs/architecture/analysis/`

**输出**：完整的分析报告文档。
