# 影响面分析器

## 目的

评估代码改动的影响范围，识别潜在风险，提供回归测试建议，避免改动引入bug或性能退化。

## 使用场景

适用于以下情况：
- 修改现有代码文件
- 重构代码结构
- 优化性能相关代码
- 修改核心系统逻辑
- 跨模块的代码改动

## 工作流程

### 阶段1：识别改动范围

**操作**：分析改动内容
- 改动的文件和方法
- 直接影响的代码
- 数据结构变化

**输出示例**：
```
改动范围：
- 文件：PlayerController.cs
- 类：PlayerController
- 方法：Update(), Move()
- 数据结构：新增 acceleration 字段
- 改动类型：核心逻辑改动
```

### 阶段2：依赖关系梳理

**操作**：分析依赖关系
- 代码依赖图谱
- 资源依赖关系
- 运行时依赖

**输出示例**：
```
代码依赖：
被引用（5个文件依赖此脚本）：
- EnemyAI.cs → 追踪玩家位置
- CameraController.cs → 跟随玩家
- UIManager.cs → 显示玩家状态

继承关系：
MonoBehaviour
  └─ PlayerController
      └─ NetPlayerController（网络版）

资源依赖：
Prefab引用：3个
- Player.prefab
- PlayerPreview.prefab

Scene引用：5个
- Level_01.unity
- Level_02.unity
```

### 阶段3：影响面评估

**操作**：评估影响程度
- 直接影响（必受影响）
- 间接影响（可能受影响）
- 潜在影响（需验证）

**输出示例**：
```
直接影响（必受影响）：
| 影响项 | 影响文件 | 影响原因 | 影响程度 |
|-------|---------|---------|---------|
| 移动手感变化 | PlayerController.cs | 移动逻辑重写 | 高 |
| 网络同步 | NetPlayerController.cs | 需同步加速度 | 中 |
| 存档兼容 | SaveManager.cs | 需保存加速度数据 | 中 |

间接影响（可能受影响）：
| 影响项 | 影响文件 | 影响原因 | 验证方法 |
|-------|---------|---------|---------|
| 敌人追踪 | EnemyAI.cs | 玩家速度变化影响追踪算法 | 实际测试 |

潜在影响：
- 教程关卡：移动教学可能需要调整
- 关卡设计：平台跳跃难度可能变化
```

### 阶段4：风险识别

**操作**：识别潜在风险
- 高风险改动点
- 兼容性问题
- 性能退化风险

**输出示例**：
```
风险识别：
| 风险点 | 风险类型 | 发生概率 | 影响程度 | 应对措施 |
|-------|---------|---------|---------|---------|
| 移动手感下降 | 功能 | 中 | 高 | 美术参与调参 |
| 存档不兼容 | 兼容性 | 中 | 高 | 版本迁移代码 |
| 网络同步延迟 | 性能 | 低 | 中 | 预测算法优化 |
| 关卡难度变化 | 功能 | 高 | 中 | 重新测试关卡 |

高风险点：
⚠️ **存档不兼容**：旧存档缺少加速度数据，需实现数据迁移
⚠️ **关卡难度变化**：所有关卡需重新测试平衡性
```

### 阶段5：测试建议

**操作**：提供测试计划
- 必须回归测试的功能
- 建议的测试用例
- 性能测试点

**输出示例**：
```
回归测试：
必须测试：
- [ ] 移动手感测试：对比改动前后手感
- [ ] 存档兼容性测试：加载旧存档
- [ ] 网络同步测试：多人模式验证

建议测试：
- [ ] 所有关卡通关测试
- [ ] 敌人AI追踪测试
- [ ] 移动教学验证

性能测试：
- [ ] 帧率测试：目标 > 60 FPS
- [ ] 内存测试：增量 < 1 MB
```

## 分析深度控制

### 快速模式
仅分析直接依赖，适用于：
- 小范围改动（< 50行）
- 非核心模块
- 时间紧急

### 标准模式
分析直接+间接依赖，适用于：
- 中等改动（50-200行）
- 核心模块
- 常规开发

### 深度模式
分析所有依赖层级，适用于：
- 大范围改动（> 200行）
- 架构核心
- 性能关键路径

## 注意事项

- 改动核心系统前**必须**执行影响面分析
- 高风险改动需要架构师审批
- 影响面分析报告作为开发文档存档
- 定期回顾分析的准确性，优化分析流程
