## 🎨 设计模式集成要点

### 架构分析时应用设计模式

AI执行架构分析时必须：
1. 查阅 `@design-patterns-overview` 了解适用模式
2. 根据问题域推荐合适的设计模式：
   - 对象创建问题 → 创建型模式（工厂、对象池）
   - 类/对象组合问题 → 结构型模式（装饰器、组合）
   - 对象通信问题 → 行为型模式（状态、观察者、命令）
3. 参考对应模式的L1级理论Skill
4. 选择对应的L2级实战Skill进行集成

### 开发流程新增设计模式选择

```
开发任务类型 → 推荐设计模式Skill：

状态管理 → @state-machine-pattern（L2级）
事件通信 → @event-system-pattern（L2级）
对象复用 → @object-pool-pattern（L2级）
输入处理 → @command-pattern（L2级）
对象创建 → @factory-method-pattern（L2级）
行为切换 → @strategy-pattern（L2级）
```
