# Unity 2D角色控制器生成器

## 目的

生成完整的Unity 2D角色移动控制器，采用渐进式披露工作流，确保代码生成前准确理解需求。

## 使用场景

适用于以下情况：
- 开发新的2D角色移动系统
- 创建平台、俯视或横版游戏的玩家控制器
- 实现角色移动功能（跳跃、冲刺、攀爬等）
- 需要生成Unity可用的角色控制器脚本

## 渐进式披露工作流

遵循**5阶段确认流程**确保精准理解：

### 阶段1：确认移动模式

**操作**：询问用户选择移动模式：
1. **平台跳跃** - 左右移动+跳跃（类似《空洞骑士》《Celeste》）
2. **俯视移动** - 四方向移动（类似《塞尔达传说》《元气骑士》）
3. **横版格斗** - 复杂状态移动（类似《街头霸王》《任天堂大乱斗》）

**确认点**：等待用户选择后再继续。

### 阶段2：确认核心功能

**操作**：根据选定模式展示必需功能和可选功能。

**平台跳跃示例**：
```
必需功能：
☑️ 左右移动
☑️ 跳跃
☑️ 地面检测
☑️ 角色翻转

可选功能：
☐ 二段跳
☐ 冲刺/闪避
☐ 攀爬
☐ 下蹲
☐ 游泳
☐ 抓墙

请指定需要哪些可选功能，或回复"默认"（仅必需功能）。
```

**确认点**：等待用户选择。

### 阶段3：确认技术方案

**操作**：展示技术选择：
1. **物理系统**：Rigidbody2D（推荐）vs 自定义物理
2. **输入系统**：新输入系统（推荐）vs 旧输入系统 vs 触屏控制
3. **动画系统**：Animator状态机 vs 程序化动画 vs 无
4. **性能优化**：标准 vs 高性能（对象池、缓存优化）

**确认点**：等待用户选择。

### 阶段4：确认移动参数

**操作**：展示默认参数并要求确认：
```
基础参数：
- 移动速度：5（范围：1-20）
- 跳跃力度：10（范围：5-30）
- 重力缩放：2（范围：1-5）

高级参数（如需调整）：
- 地面检测距离：0.1
- 加速度：10
- 减速度：10

确认参数或指定调整值。
```

**确认点**：等待用户确认。

### 阶段5：生成代码

**操作**：生成完整的PlayerController.cs脚本，包含：
- ✅ 流畅的移动控制
- ✅ 精准的跳跃与地面检测
- ✅ 用户选择的可选功能
- ✅ Gizmos调试可视化
- ✅ 完整的XML文档
- ✅ 符合Unity最佳实践

**输出**：提供生成的代码和使用说明。

## 核心原则

### ✅ 不假设
- 不猜测用户需求
- 不猜测技术选择
- 不猜测参数值

### ✅ 主动确认
- 每个阶段都询问
- 提供清晰的选项
- 等待明确确认

### ✅ 渐进披露
- 不一次性展示所有复杂度
- 按需披露高级选项
- 用户掌控开发节奏

## 代码规范

生成的代码遵循：
- 私有字段使用`[SerializeField]`
- 添加`[Tooltip]`特性说明
- 使用`#region`划分代码区域
- 在`Awake()`中缓存组件引用
- 物理操作放在`FixedUpdate()`
- 完整的XML注释
- 符合Unity最佳实践

## 验收标准

- [ ] 编译无错误无警告
- [ ] 移动流畅无抖动
- [ ] 跳跃高度合理（平台模式）
- [ ] 翻转及时正确
- [ ] 动画参数正确传递（如启用）
- [ ] 包含Gizmos调试绘制
- [ ] 代码文档完整
