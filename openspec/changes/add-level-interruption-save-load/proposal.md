## Why

当前设计关卡在玩家中断（关闭游戏、切换场景、崩溃）时无法保存进行中的状态，导致关卡配置和进度丢失、重复操作成本高。随着关卡内容和编辑步骤增多，需要尽快提供可靠的存读档能力来保障编辑连续性与数据安全。

## What Changes

- 新增“设计关卡中断存档”能力：在关键节点（手动保存、退出关卡、应用暂停/退出）将当前关卡编辑状态持久化到本地。
- 新增“设计关卡恢复读档”能力：重新进入设计关卡时检测可恢复档案，并支持继续上次编辑或放弃恢复。
- 定义存档数据结构与版本字段，支持未来字段扩展和兼容处理。
- 增加异常处理策略：存档损坏、版本不兼容、写入失败时给出降级行为与提示，不阻断基础关卡编辑流程。

## Capabilities

### New Capabilities
- `level-interruption-save-load`: 设计关卡在中断场景下的本地存档写入、恢复读取、兼容校验与恢复决策流程。

### Modified Capabilities
- 无

## Impact

- 受影响系统：设计关卡编辑流程、关卡状态管理、应用生命周期事件处理（暂停/退出）与本地持久化模块。
- 受影响数据：关卡布局、编辑操作上下文、恢复元信息（时间戳、版本、校验状态）。
- 对外 API 影响：无公开 API 变更；主要为内部模块协议与数据结构新增。
