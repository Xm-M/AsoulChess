# 渐进式披露开发规范

## 规范目的

确保AI在未知背景下主动寻求用户确认，避免假设和猜测，实现精准需求理解和可控开发。

## 核心原则

### 1. 不假设原则

**禁止假设**：
- ❌ 不假设用户需求细节
- ❌ 不假设技术实现方案
- ❌ 不假设用户偏好

**正确做法**：
- ✅ 明确不确定的信息
- ✅ 主动询问澄清
- ✅ 等待用户确认

### 2. 主动确认原则

**强制要求**：
- ✅ 不确定时立即询问
- ✅ 提供选项让用户选择
- ✅ 等待确认再执行

**禁止行为**：
- ❌ 未确认需求直接生成代码
- ❌ 未提供选项直接选择默认值
- ❌ 超出确认范围的功能开发

### 3. 分步披露原则

**信息分级**：
1. **第一层**：核心需求（必须确认）
2. **第二层**：技术方案（按需确认）
3. **第三层**：实现细节（可选确认）

**披露策略**：
- ✅ 不一次性展示所有复杂度
- ✅ 按需披露实现细节
- ✅ 逐步深入配置选项

## 执行要求

### AI行为规范

#### 第一步：识别信息缺口

**必须确认的信息类别**：

**1. 需求确认**
- ❓ 功能的具体目标是什么？
- ❓ 有没有参考案例？
- ❓ 是否有特殊约束？
- ❓ 优先级如何？

**2. 技术细节确认**
- ❓ 使用什么技术方案？
- ❓ 是否需要性能优化？
- ❓ 是否需要兼容现有系统？
- ❓ 有无特定技术偏好？

**3. 设计决策确认**
- ❓ 使用什么设计模式？
- ❓ 数据如何组织？
- ❓ 接口如何定义？
- ❓ 是否需要扩展性？

**4. 实现细节确认**
- ❓ 命名约定？
- ❓ 代码风格？
- ❓ 注释程度？
- ❓ 测试要求？

#### 第二步：提出澄清问题

**标准格式**：
```
❓ [问题类别]

[问题描述]

选项：
1. [选项1]
2. [选项2]
3. [选项3]

或者告诉我您的具体需求。
```

#### 第三步：等待用户确认

**等待规则**：
- ✅ 必须等待用户回复
- ✅ 收到完整信息后再执行
- ❌ 不得猜测或假设用户意图

#### 第四步：执行开发任务

**执行前提**：
- ✅ 信息缺口已全部填补
- ✅ 用户已明确确认
- ✅ 范围清晰可控

## 违规判定

### 严重违规

**立即纠正**：
1. 未确认需求直接生成代码
2. 未提供选项直接选择默认值
3. 超出确认范围的功能开发

**处理方式**：
- 立即停止当前操作
- 回退到确认环节
- 重新执行渐进式披露

### 一般违规

**需改进**：
1. 提供的选项不充分
2. 问题描述不清晰
3. 确认流程不完整

**处理方式**：
- 补充缺失的确认环节
- 优化问题描述格式
- 完善确认流程

## 使用示例

### 正确示例

```
用户: 帮我生成一个背包系统

AI: 好的！在开始之前，我需要确认几个关键信息：

📦 背包类型：
1. 网格背包（类似《暗黑破坏神》）
2. 列表背包（类似《原神》）
3. 混合背包（网格+列表）

请问您需要哪种类型？

---

用户: 网格背包，类似《我的世界》

AI: 明白了！网格背包，类似《我的世界》风格。

接下来确认几个技术细节：

1. 📐 网格尺寸：
   - 固定大小（如 8x5）
   - 动态扩展

2. 🎮 物品交互：
   - 需要拖拽移动
   - 需要堆叠合并
   - 需要快捷使用

请告诉我您的需求，或使用默认配置（固定8x5网格，支持拖拽和堆叠）。

---

用户: 默认配置就可以

AI: 完美！开始生成网格背包系统...

[开始生成代码]
```

### 错误示例

```
用户: 帮我生成一个背包系统

AI: [直接生成完整代码，未确认任何细节]
using UnityEngine;
using System.Collections.Generic;

public class InventorySystem : MonoBehaviour
{
    // ... 200行代码
}
```

**问题**：
- ❌ 未确认背包类型
- ❌ 未确认功能需求
- ❌ 未确认技术方案
- ❌ 未确认参数配置

## 渐进式披露流程图

```
用户提出需求
    ↓
AI识别信息缺口
    ↓
AI提出澄清问题 ←──┐
    ↓               │
用户回答           │
    ↓               │
信息是否完整？ ────┘
    │ 是
    ↓
AI执行任务
```

## 特殊情况处理

### 用户明确指定参数

**情况**：用户在需求中已明确所有参数

**处理**：
```
用户: @unity-2d-character
模式: platform
跳跃: yes
动画: yes
速度: 6

AI: 收到！直接按指定参数生成：

✓ 模式：平台跳跃
✓ 功能：移动 + 跳跃 + 动画
✓ 参数：速度6

正在生成代码...
```

### 紧急情况

**情况**：时间紧急，需要快速开发

**处理**：
- 使用快速模式，提供默认选项
- 但必须告知用户："使用默认配置，如有问题请及时反馈"
- 开发后必须进行充分测试
